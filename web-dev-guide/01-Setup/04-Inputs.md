# Analysis Inputs

## Inputs You Will Receive

You will receive in this or similar order:

1. **Analysis metadata**.
   - Project details
   - Client information
   - Audit reason and scope

2. **Original documentation or prompt**, which defines:
   - Scope.
   - Functional and non-functional requirements.
   - Acceptance criteria (if they exist).

3. **Source code / document(s)** of the project(s).
   - Complete codebase
   - Project files and structure

4. **Lists of specific requirements for the integration of selected technologies**.
   - Technology-specific checklists
   - Requirements for each applicable technology

## Usage

You must use **all** this context to generate your checklist.
